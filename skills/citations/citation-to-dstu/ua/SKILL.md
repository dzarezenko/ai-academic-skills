# Skill: Конвертація посилань → ДСТУ 8302:2015

## Призначення

Цей скілл конвертує бібліографічні посилання з міжнародних стилів цитування
(APA, Chicago, MLA, IEEE, Vancouver, Harvard, BibTeX тощо)
у формат ДСТУ 8302:2015 (національний стандарт бібліографічного опису України).

Результат має відповідати:
ДСТУ 8302:2015 + типовим вимогам українського академічного оформлення.

---

## Підтримувані вхідні стандарти (формати)

Скілл має коректно інтерпретувати та нормалізувати:

- APA (6th, 7th edition)
- Chicago (Notes & Bibliography, Author-Date)
- MLA (8th, 9th)
- IEEE
- Vancouver
- Harvard
- BibTeX
- ISO 690
- AMA
- Turabian
- ACS
- Bluebook
- ГОСТ та пострадянські “гібридні” формати
- Nature style
- Springer / Elsevier журнальні формати
- arXiv-референси
- формати посилань у тезах/збірниках конференцій
- DOI-only посилання
- URL-only посилання
- змішані / частково зламані / неповні записи

---

## Цільовий стандарт

ДСТУ 8302:2015
«Інформація та документація. Бібліографічне посилання. Загальні положення та правила складання».

---

## Базові правила форматування ДСТУ

### 1) Автори

Формат:
Прізвище Ініціали.

Приклад:
Sun J., Yao X., Wang S., Wu Y.

Якщо авторів більше трьох:
за замовчуванням виводити всіх авторів (без “et al.”), якщо не задано іншої політики.

---

### 2) Стаття у журналі

Формат:

Автор(и). Назва статті. Назва журналу. Рік. Том(номер). С. сторінки.
DOI або URL (за наявності). (дата звернення: DD.MM.YYYY) / (accessed: DD.MM.YYYY).

Приклад (англомовне джерело):

Sun J., Yao X., Wang S., Wu Y. Blockchain-Based Secure Storage and Access Scheme for Electronic Medical Records in IPFS. IEEE Access. 2020. Vol. 8. P. 59389–59401. URL: https://doi.org/xxxxx (accessed: 19.10.2024).

---

### 3) Книга

Автор(и). Назва : підзаголовок. Видання (за потреби). Місце: Видавництво, Рік. К-сть с.

---

### 4) Матеріали конференції

Автор(и). Назва. У: Назва конференції. Місце проведення, дати. С. сторінки.

---

### 5) Електронний ресурс (онлайн-джерело)

Назва. URL: посилання (дата звернення: DD.MM.YYYY).

---

### 6) Збереження мови та оригінальності

- Не перекладати назви праць автоматично.
- Зберігати оригінальну капіталізацію та орфографію.
- Україномовні джерела — українська типографіка.
- Англомовні — англійська типографіка.

---

### 7) Формат дати звернення

Для україномовних записів:
(дата звернення: DD.MM.YYYY)

Для англомовних записів:
(accessed: DD.MM.YYYY)

---

### 8) Том / номер / сторінки

Використовувати скорочення:
Vol., No., P., С.

Не використовувати:
pp., Issue, # (як елемент стилю)

---

### 9) Пріоритет DOI

Якщо є DOI — виводити його як основний ідентифікатор (можна у вигляді URL https://doi.org/...).
URL сторінки допускається додатково лише якщо це потрібно і не дублює DOI.

---

### 10) Порядок елементів і пунктуація

- Крапка після блоку авторів.
- Крапка після назви.
- Крапка після назви журналу/джерела.
- Рік — з крапкою.
- Пробіл після знаків пунктуації.
- Для діапазону сторінок — тире (en dash) “–”.

---

## Приклади input → output

### Приклад 1: APA → ДСТУ (стаття у журналі)

**Input (APA 7th):**

```
Sun, J., Yao, X., Wang, S., & Wu, Y. (2020). Blockchain-based secure storage and access scheme for electronic medical records in IPFS. IEEE Access, 8, 59389–59401. https://doi.org/10.1109/ACCESS.2020.2982964
```

**Output (ДСТУ 8302:2015):**

```
Sun J., Yao X., Wang S., Wu Y. Blockchain-Based Secure Storage and Access Scheme for Electronic Medical Records in IPFS. IEEE Access. 2020. Vol. 8. P. 59389–59401. URL: https://doi.org/10.1109/ACCESS.2020.2982964 (accessed: 19.10.2024).
```

### Приклад 2: BibTeX → ДСТУ (матеріали конференції)

**Input (BibTeX):**

```
@inproceedings{smith2019deep,
  author    = {Smith, John and Lee, Anna},
  title     = {Deep Learning for Natural Language Processing},
  booktitle = {Proceedings of the 2019 Conference on AI},
  year      = {2019},
  pages     = {112--120},
  address   = {New York, USA}
}
```

**Output (ДСТУ 8302:2015):**

```
Smith J., Lee A. Deep Learning for Natural Language Processing. Proceedings of the 2019 Conference on AI. New York, USA, 2019. P. 112–120.
```

### Приклад 3: Україномовне джерело (ГОСТ → ДСТУ)

**Input (ГОСТ-подібний):**

```
Шевченко І. В. Основи інформаційних технологій : підручник / І. В. Шевченко. — К. : Наукова думка, 2018. — 320 с.
```

**Output (ДСТУ 8302:2015):**

```
Шевченко І. В. Основи інформаційних технологій : підручник. Київ : Наукова думка, 2018. 320 с.
```

## Логіка конвертації

1. Визначити/припустити вхідний стиль (за структурою).
2. Розпарсити метадані:
   - автори
   - назва
   - контейнер (журнал/книга/збірник)
   - рік
   - том
   - номер
   - сторінки
   - DOI / URL
   - дату звернення (якщо є)
3. Нормалізувати авторів.
4. Нормалізувати пунктуацію та скорочення.
5. Застосувати шаблон ДСТУ для відповідного типу джерела.
6. Перевірити чекліст відповідності.
7. Повернути один форматований бібліографічний запис.

---

## Edge cases (граничні випадки)

- **DOI-only посилання** — якщо вхід містить лише DOI (напр. `10.1109/ACCESS.2020.2982964`), спробувати отримати метадані та сформувати повний запис ДСТУ. Якщо не вдалося — повернути DOI у форматі URL.
- **URL-only посилання** — оформити як електронний ресурс з датою звернення.
- **Неповні записи** — якщо відсутній рік, том або сторінки, пропустити відповідний елемент, але зберегти коректну пунктуацію ДСТУ.
- **Змішана мова** — якщо запис містить і кирилицю, і латиницю, визначити основну мову за назвою праці та застосувати відповідну типографіку.
- **Більше трьох авторів** — за замовчуванням виводити всіх; не скорочувати до "та ін." / "et al." без явної вказівки.
- **Скорочені назви міст** — розгортати загальновідомі скорочення: К. → Київ, М. → Москва, СПб. → Санкт-Петербург.
- **Дублювання автора (ГОСТ-стиль)** — прибирати повтор автора після косої риски (`/ І. В. Шевченко`).
- **Тире замість en dash** — замінювати дефіс (-) та em dash (—) на en dash (–) у діапазонах сторінок.
